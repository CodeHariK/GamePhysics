# SAT Comparison Analysis: Physics Engine vs. Box2D Lite

The Separating Axis Theorem (SAT) implementation in [SAT.ts](../../lib/collision/SAT.ts) is a standard baseline, but Box2D Lite (and the full Box2D) uses a more refined approach that is better for physics stability.

---

## ⚖️ Efficiency Comparison

| Feature | Current `SAT.ts` | Box2D Lite | Matter.js |
| :--- | :--- | :--- | :--- |
| **Normal Calculation** | Dynamically calculated every frame for both A and B edges. | Usually pre-calculated/transformed. | Fast lookup of pre-calculated `axes`. |
| **Projection Logic** | $O(N \cdot (N+M))$ | $O(N+M)$ | $O(N+M)$ |
| **Contact Point Method**| Point-in-Poly heuristic. | **Sutherland-Hodgman Clipping**. | **Hill-climbing & Containment**. |
| **Stability** | Moderate | **High** (Best for stacking). | **High** (General purpose). |

---

## 🏗️ Stability Analysis (The Core Difference)

### 1. Contact Manifold (Clipping vs. Point-In-Poly)
- **SimpleSAT (Point-in-Poly)**: This is a **heuristic**. It only returns the **existing vertices** of the polygons that happen to be inside the other shape. If a flat edge is resting on another, it might find 0, 1, or 2 vertices. If it finds 0, it falls back to a center-point average. This causes the "jitter" and unstable stacking you see in many basic physics engines.
- **RobustSAT (Clipping)**: This is a **geometric intersection**. It identifies the "Reference Face" and the "Incident Face" and then mathematically **clips** the incident edge against the reference planes. 
- **The Result**: Clipping often creates **new points** that did not exist in the original polygons (where the edges intersect). RobustSAT finds the *complete segment* of overlap, ensuring a stable "locked-in" feel for the physics solver.

### 2. Deep Intersections
Standard SAT can struggle with very deep intersections or extremely high speeds. Box2D Lite's clipping logic is more deterministic in how it resolves the "best" point of contact.

### 3. Accumulated Impulses (Warm Starting)
While not strictly part of SAT, Box2D Lite uses iterative solvers that benefit massively from the **Consistent Manifold** generated by clipping. By providing fixed contact points from frame to frame, the engine can "remember" the forces it applied earlier.

---

> [!TIP]
> **Summary Recommendation**
> If you plan to build stacks of objects or need high-precision collision (e.g., a platformer with many small blocks), using the **Clipping Manifold** logic Is a significant upgrade. It provides the "Feature ID" necessary for professional physics stability.
